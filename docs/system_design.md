# システム定義・業務フロー定義書

本ドキュメントでは、**SalesPlan Pro（販売計画管理システム）**の定義、役割、および業務における利用フローについて記述します。

## 1. システム定義と役割

### システムの目的
本システムは、小売業における**数値計画（予算・販売計画）策定業務の効率化と高度化**を目的としています。
従来Excel等で分散管理されていた計画データを一元化し、精緻な予実管理と迅速な意思決定（PDCAサイクルの高速化）を実現します。

### システムの役割・位置付け
本システムは、基幹システムや外部情報システムと連携し、計画策定の「ハブ」として機能します。

*   **Input**: 外部システムからのマスタ情報（商品・店舗・原価）、過去の実績データ。
*   **Process**: 階層別（商品・店舗）の計画数値入力、シミュレーション、承認フロー。
*   **Output**: 確定予算データ、予実差異分析レポート、リカバリー施策の立案支援。

## 2. 業務フロー (Operational Workflow)

提示いただいた月間/週間販売計画業務フローに基づき、本システムでのプロセスを定義します。

### 業務フロー図 (Mermaid)

```mermaid
graph TD
    %% レーン定義
    subgraph Data [データソース/連携]
        PreResults[前月/前週実績]
        Master[商品/店舗マスタ]
    end

    subgraph User_MD [MD/BY (計画策定)]
        M1[M1 販売実績確認]
        M2[M2 戦略/定常 商品改廃]
        
        subgraph Planning [詳細計画策定]
            M3[M3 基準売価設定/更新]
            M4[M4 商品目標金額調整]
            M5[M5 売上/粗利確認 Rep.]
        end
        
        Meeting[販売会議 (修正・確定)]
        
        subgraph Registration [システム登録]
            M6[M6 戦略/定常登録]
            M7[M7 商品目標金額確定]
            Report[月次/週次販売計画 Rep.]
        end
    end

    subgraph User_Mgr [BULeader/部長 (承認)]
        Approval{M8 計画確認・承認}
    end

    %% フロー接続
    PreResults --> M1
    Master --> M1
    M1 --> M2
    
    M2 -->|変更あり| M3
    M2 -->|変更なし| M4
    
    M3 --> M4
    M4 --> M5
    M5 --> Meeting
    
    Meeting -->|要修正| M4
    Meeting -->|確定| M6
    
    M6 --> M7
    M7 --> Report
    Report --> Approval
    
    Approval -->|NG/差戻| Meeting
    Approval -->|OK/完了| End((計画確定))

    %% システム支援機能
    Sys_Support[自動格上提案/予測シミュレーション] -.-> M2
    Sys_Support -.-> M4
```

### 各ステップの詳細

#### 1. 現状分析・準備 (M1, M2)
*   **M1 販売実績確認**: 前月（または前週）の販売実績レポートを確認し、達成状況や課題を把握します。
*   **M2 戦略/定常 商品改廃**: 実績に基づき、商品の「戦略（売り込み）」⇔「定常（定番）」区分の見直し（格上げ/格下げ）を行います。システムは実績から候補を自動提案します。

#### 2. 計画策定・シミュレーション (M3, M4, M5)
*   **M3 基準売価設定**: 戦略商品や市況変化に伴う売価変更がある場合、マスタの基準売価を更新します。
*   **M4 商品目標金額調整**: 商品階層（カテゴリ/SKU）ごとに、売上・粗利・数量の目標値を調整します。
*   **M5 売上/粗利確認**: シミュレーション結果（着地見込）を確認します。

#### 3. 調整・合意 (販売会議)
*   策定した計画案をもとに、MD/BY間、または上長と販売会議を行い、方針のすり合わせと修正を行います。

#### 4. 登録・承認 (M6, M7, M8)
*   **M6 戦略/定常登録**: 確定した商品の区分（戦略/定常）をシステムに登録します。
*   **M7 商品目標金額確定**: 最終的な目標数値をシステム上で確定（保存）します。
*   **M8 計画確認・承認**: BULeader（販売部長）がシステム上で計画を確認し、最終承認を行います。

## 3. 主要機能と利用シーン

| フローID | 機能名 | 利用者 | 主な利用シーン | 特徴 |
|---|---|---|---|---|
| **M1** | **ダッシュボード / 実績分析** | MD/BY | 毎朝・週初の状況確認 | 直近1ヶ月/1週間の実績によりチャンスロスや好調商品を特定。 |
| **M2, M6** | **商品階層別計画 (属性設定)** | MD/BY | 商品改廃・区分変更 | チェックボックス等で「戦略」「定常」区分を切り替え、自動提案機能の活用。 |
| **M3, M4** | **商品階層別計画 (数値入力)** | MD/BY | 予算入力・シミュレーション | 基準売価や数量を変更し、売上・粗利への影響をリアルタイムに確認。 |
| **M5** | **計画レポート出力** | MD/BY | 販売会議資料作成 | 計画数値をExcel/PDF等でレポート出力し、会議での検討資料とする。 |
| **M8** | **承認ワークフロー** | 部長 | 計画の最終承認 | 申請された計画の内容査読、コメント付き承認・差戻し。 |

## 4. データの取り扱いについて

### マスタデータの優先順位
システム内で使用する「原価」「売価」などのマスタ情報は、**外部システム（生鮮商談システムなど）を正**とします。本システム上で一時的にシミュレーション値を入力することも可能ですが、最終的な発注や価格マスタへの反映は外部システムを通じて行われます。

### データ更新頻度
*   **マスタ**: 日次（毎朝6:00連携）
*   **実績**: 日次（前日分を毎朝9:30連携）およびリアルタイム（速報値）
*   **計画**: 随時（ユーザーによる保存・更新）
